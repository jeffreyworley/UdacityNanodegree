Basic Concepts of HTML

Lesson 1.1
WWW or the World Wide Web:

 The World Wide web is a collection of HTML (Hypertext markup language) documents. However, any type of file can exist on the web. The web works by using hyperlinks to connect HTML files to other HTML files.

Major Pieces of the Internet

 There are a few main pieces that are required to connect you to the internet. these are you, your computer, your browser, internet connection, and servers. you make a request to your computer on your browser to access a web page. Your browser then connects to the server that hosts the web page you want to access through the internet. The data is transfered to your computer where it is displayed on your browser.

HTML

 HTML is made up of text content, markup, references to other documents, and links to other pages. HTML markup consists of tags that are used to change and manipulate the text and control the aspects of the document.The references to other documents and links to other pages are what makes the internet, the internet. It connects all the web pages together for a user to access.

HTML Tags

HTML Tags look like this:
 tag name> contents </tag name>

 Tags serve many purposes such as italicizing text, bolding text, creating hyperlinks on text, and any other text manipulation. Tags also can include links for pictures to render in the HTML document.

Computers are stupid

 I, as many other programmers, have experienced the unfortunate issue of mistyping a keyword or forgetting a semicolon, or something along those lines that makes a computer do something you didn't want it to do. However, whatever the mistake was, told the computer to do just what it did. A computer has no way of understanding what you meant to say. It can only understand exactly what you instruct it to do.

Inline VS. Block elements

 Withing HTML tags, there is a distinction between inline tags, and block tags. Inline tags simply change the text without affecting text around it. Block tags, however, create a padded "box" around the contents of the text.

CSS

 Cascading style sheet. it is what defines the “styles” of applied tags. does the most specific instruction or rule. styles are defined by classes or by tags.

CSS Style

 Changing the visual properties of an element only changes how the element is displayed. However it does not change the kind of element. So, use the correct tag for what you need the text to be. i.e. use a h1 tag for a header etc.

 Changing the tag changes the semantics of the content and the look of the content.

The Box Model

 CSS has several properties. some of these include the Margin, border padding, and content. These are what can be changed and redefined to determine the box that content will sit in inside of a webpage. The following explains the parts of the box model in depth.

Content

 Image or text that is appearing on the website. Kind of like the nucleus of the box model. The rest of the properties exist to “protect” the content from the rest of the components on the webpage. Element width is defined as the width including border, padding, and content.

Padding

 Clears an area around the content and is effected by the background color of the box. Space inside of the box that protects the content directly.

Border

 Goes around padding and content. Inherited from color property of the box.

Margin

 Clears the area around the border. No background color. completely transparent. basically the spacer between boxes.

Constant Box Sizing

 by using the code box-sizing: border-box; you can always have a constant for the amount of pixels an element will span across not including margin. the only issue is that some browsers are to old to support this. So, you add browser specific prefixes. webkit, most, ms. The size of box is sometimes calculated in pixels, but you can also use a percentage. Percentage allows for a better user experience.
